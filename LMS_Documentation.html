🏫 1. Project Overview

Saeed MDCAT LMS is a full-stack Learning Management System built for MDCAT (Medical & Dental College Admission Test) preparation. It supports three user roles and provides test-taking, MCQ management, course content, and a reporting/feedback system.

Technology Stack
LayerTechnology
RuntimeNode.js (Express.js)
DatabaseMongoDB (Mongoose ODM)
AuthenticationJWT (Access + Refresh) + Google OAuth 2.0
FrontendReact 18 + React Router v6
StylingTailwindCSS
HTTP ClientAxios (with interceptors)
File HandlingMulter (local uploads)
Document Parsingmammoth / docx parser
User Roles
RoleCapabilities
studentTake tests, save questions, report MCQs, view courses, generate auto-tests
teacherCreate/manage tests & MCQs, import documents, respond to reports, manage QBs
adminFull access: users, courses, QBs, reports, settings, all teacher features

🛡️ 13. Middleware

protect (auth.js)

  • Reads Authorization: Bearer {token} header OR accessToken cookie
  • Verifies token using JWT_ACCESS_SECRET
  • Returns 401 with tokenExpired: true if expired
  • Returns 401 if token is invalid or missing
  • Attaches req.user with user document from DB

authorize(...roles) (roleCheck.js)

  • Called as authorize('admin') or authorize('admin','teacher')
  • Checks req.user.role against allowed roles array
  • Returns 403 Forbidden if role not in allowed list
  • Always used after protect middleware

🧠 15. Controllers & Business Logic

authController.js

FunctionWhat It Does
register()Validates input → hashes password → creates User (role=student) → calls sendTokenResponse()
login()Finds user by email → matchPassword() → calls sendTokenResponse()
googleAuthGIS()Calls Google tokeninfo API → finds or creates user → sendTokenResponse()
getMe()Returns req.user (attached by protect middleware)
logout()Clears user.refreshToken in DB → sets expired cookies
refreshToken()Reads refreshToken cookie → jwt.verify() → issues new accessToken. Used by Axios response interceptor
sendTokenResponse()Helper: calls getSignedAccessToken() + getSignedRefreshToken() → sets cookies → returns JSON with {success, accessToken, user}

userTestController.js (Test Taking Engine)

FunctionLogic
startTest()Checks test is published → checks for existing in-progress attempt (resume) → creates new UserTestAttempt with questionAttempts array pre-filled with all MCQ IDs → returns attempt
submitAnswer()Updates questionAttempts[questionIndex].selectedOption → looks up correct option from MCQ → sets isCorrect → returns result in tutor mode
reportQuestion()Sets questionAttempts[questionIndex].reported = true → creates MCQReport document → returns updated attempt
completeTest()Sets status='completed', endTime, totalTimeSpent → pre-save hook recalculates score, maxScore, scorePercentage

mcqController.js

FunctionLogic
createMCQ()Validates at least one option isCorrect → creates MCQ → if testId present: pushes to test.mcqs, increments test.totalQuestions
updateMCQ()Updates MCQ fields → increments revisionCount → sets lastRevised = Date.now()
deleteMCQ()Deletes MCQ → runs Test.updateMany({mcqs: mcqId}, {$pull: {mcqs: mcqId}}) + decrements totalQuestions
uploadDocument()Multer stores .docx in temp → calls extractMCQsFromDoc() async → creates import status record → returns importId immediately (non-blocking)
getTopicCounts()MongoDB $group aggregation: { _id: '$qbTopicId', count: { $sum: 1 } }

questionBankController.js — generateTest()

Auto-Test Generation Logic:
1. Build filter query (QB + optional subjectIds/chapterIds/topicIds)
2. MCQ.aggregate([{$match: filter}, {$sample: {size: count}}]) — random sample
3. If existingTestId provided: fetch existing test, add new MCQ IDs to test.mcqs, update totalQuestions
4. Otherwise: create new Test with all metadata, set isPublished: true, status: 'published'
5. Return created/updated test with populated MCQs

mcqReportController.js — Key Patterns

FULL_POPULATE constant (used on all report reads)
const FULL_POPULATE = [
  { path: 'mcq', select: 'questionText options
    subject unit topic difficulty explanationText

  { path: 'reportedBy', select: 'fullName email role' },

getCountsByMcqs() — batch aggregation
MCQReport.aggregate([
  { $match: {
    mcq: { $in: ids },
    status: { $in: ['open','active'] }
  }},
  { $group: {
    _id: '$mcq',

⚛️ 16. Frontend Architecture
API Configuration (axiosConfig.js)
Base URL: http://localhost:5000/api
         (or REACT_APP_API_URL env var)
withCredentials: true  // enables cookies
Request Interceptor
  • Reads accessToken from localStorage
  • Adds Authorization: Bearer {token} header
Response Interceptor (auto token refresh)
  • On 401 with tokenExpired: true flag from backend
  • Calls POST /auth/refresh-token
  • Updates localStorage with new accessToken
  • Retries original failed request automatically
  • On refresh failure: clears localStorage → redirects to /login
Auth Context & Hook
  • AuthContext — wraps app, provides user state
  • useAuth() hook exposes: user, isAuthenticated, isAdmin, isTeacher, isStudent
  • Methods: login(), register(), logout(), loginWithGoogle()
  • Guard components: PrivateRoute (any auth) + RoleBasedRoute (specific roles)
Auth Flow
  • Login → accessToken stored in localStorage (1hr)
  • refreshToken stored in HttpOnly cookie (1yr)
  • Axios auto-refreshes on token expiry silently
  • Google OAuth via @react-oauth/google library

MCQ Document Import Flow
  • Frontend uploads .docx + testId/questionBankId
  • Backend stores in temp dir → returns importId immediately
  • Async: extractMCQsFromDoc() parses questions, options, answers, images from .docx
  • Images embedded in .docx are extracted → saved to uploads/images/
  • Frontend polls GET /mcqs/import-status/{importId} every 2 seconds
  • Status transitions: processing → completed|error

✅ 21. Feature Summary by Role

Student Features

  • Take tests in Tutor mode (immediate answer feedback) or Timer mode (timed exam)
  • Save questions for later review with notes & tags
  • Report problematic MCQs during tests
  • View conversation thread with teacher/admin on reports
  • Submit satisfaction feedback after report resolution
  • View full test history with scores
  • Review past test attempts question by question
  • Browse course catalog and view resources/notes/PDFs
  • Generate auto-tests from any Question Bank
  • Filter reports by QB, status, satisfaction

Teacher Features

  • Create and manage tests with full metadata
  • Create MCQs individually with rich text editor
  • Bulk import MCQs from .docx documents
  • Manually pick MCQs from Question Bank into test
  • Edit all MCQs sequentially with keyboard shortcuts (Alt+←/→)
  • View reports assigned to them, respond to students
  • Assign themselves to open reports
  • View own report handling statistics
  • See report count badges on MCQ list and sequential editor
  • Edit MCQ directly from report card link

Admin Features

  • All teacher features
  • Full user management (create/edit/delete, any role)
  • Create and manage courses with subject/chapter hierarchy
  • Upload course cover images and PDF notes
  • Create and manage Question Banks with hierarchy
  • Import MCQs into Question Banks from .docx
  • View all reports from all teachers and students
  • View teacher performance statistics dashboard
  • Configure system settings (max MCQs, default QB)
  • Generate auto-tests for any user context

Auto-Test Generation

  • Select Question Bank
  • Filter by subject / chapter / topic
  • Set MCQ count (limited by system settings)
  • Configure title, duration, passing score, difficulty
  • MongoDB $sample for random selection
  • Create new test OR add to existing test
  • Test published immediately and ready for students
